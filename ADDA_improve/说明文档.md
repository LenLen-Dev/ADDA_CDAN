# PL-EPA-ADDA：融合伪标签、熵最小化与类原型对齐的 ADDA 改进版

本目录是在原始 ADDA（Adversarial Discriminative Domain Adaptation）基础上实现的改进版本。原始 ADDA 的核心是：先用有标签源域训练源分类器，再复制一份目标特征提取器，通过域判别器让目标特征分布接近冻结的源特征分布。

## 1. 原始 ADDA 思想

单源无监督域自适应中有：

- 源域：有标签样本 \((x_s, y_s)\)；
- 目标域：无标签样本 \(x_t\)。

ADDA 分两阶段训练：

1. **源域监督预训练**：训练源编码器/分类器，使其能正确分类源域样本。
2. **目标域对抗适配**：复制源模型得到目标模型，冻结源模型，训练目标编码器欺骗域判别器，使目标特征看起来像源域特征。

优势是训练流程清晰、实现简单；不足是对抗对齐主要约束整体边缘分布，容易出现类别错配。例如目标域某一类可能被对齐到源域另一类附近。

## 2. 为什么可以改进

可以改进。原因是 ADDA 只依赖域判别器做全局分布对齐，没有充分利用目标预测的类别结构。针对这一点，本实现加入三类语义约束：

1. **伪标签（Pseudo Label）**：对高置信度目标样本生成伪标签，用交叉熵强化类别判别边界。
2. **熵最小化（Entropy Minimization）**：降低目标预测分布熵，让目标样本远离分类边界。
3. **类原型对齐（Class Prototype Alignment）**：用源域有标签特征维护每一类的源原型，并把高置信度目标特征拉向其伪标签对应的源类原型，缓解类别错配。

## 3. 总体损失

保留 ADDA 的域对抗损失：

\[
\mathcal{L}_{adv}=\mathcal{L}_{D}(F_s(x_s), F_t(x_t))
\]

新增目标侧语义损失：

\[
\mathcal{L}_{pl}=CE(C(F_t(x_t)), \hat y_t),\quad \max p_t \ge \tau
\]

\[
\mathcal{L}_{ent}=-\sum_c p_{t,c}\log p_{t,c}
\]

\[
\mathcal{L}_{proto}=1-\cos(F_t(x_t), \mu^s_{\hat y_t})
\]

总损失：

\[
\mathcal{L}=\lambda_{adv}\mathcal{L}_{adv}
+ r(e)(\lambda_{pl}\mathcal{L}_{pl}
+ \lambda_{ent}\mathcal{L}_{ent}
+ \lambda_{proto}\mathcal{L}_{proto})
\]

其中 \(r(e)\) 是线性 warm-up，避免训练早期伪标签质量低时过强约束目标特征。

## 6. 运行示例

Office31 示例：

```bash
python ADDA_improve/adda_improve.py examples/domain_adaptation/image_classification/data/office31 \
  -d Office31 -s A -t W -a resnet18 --epochs 20 --pretrain-epochs 3 \
  --pseudo-threshold 0.95 --pseudo-label-weight 0.1 \
  --entropy-weight 0.01 --prototype-weight 0.1 \
  --regularizer-start-epoch 1 --regularizer-rampup-epochs 5 \
  --log logs/pl_epa_adda/Office31_A2W
```

Digits 小规模示例：

```bash
python ADDA_improve/adda_improve.py examples/domain_adaptation/image_classification/data/digits \
  -d Digits -s MNIST -t USPS --train-resizing res. --val-resizing res. \
  --resize-size 28 --no-hflip --norm-mean 0.5 --norm-std 0.5 \
  -a lenet --no-pool --scratch -b 32 -i 100 --epochs 5 \
  --pseudo-threshold 0.9 --log logs/pl_epa_adda/MNIST2USPS
```

## 7. 主要参数

| 参数 | 默认值 | 含义 |
| --- | --- | --- |
| `--trade-off` | `1.0` | ADDA 域对抗损失权重 |
| `--pseudo-threshold` | `0.95` | 目标伪标签置信度阈值 |
| `--pseudo-label-weight` | `0.1` | 伪标签交叉熵权重 |
| `--entropy-weight` | `0.01` | 目标熵最小化权重 |
| `--prototype-weight` | `0.1` | 类原型对齐权重 |
| `--prototype-momentum` | `0.9` | 源类原型 EMA 动量 |
| `--regularizer-start-epoch` | `1.0` | 从第几个 epoch 开始启用目标语义约束 |
| `--regularizer-rampup-epochs` | `5.0` | 目标语义约束线性 warm-up 长度 |
| `--optimize-head` | 关闭 | 是否在适配阶段也更新目标分类头；默认保持 ADDA 的冻结分类头思想 |

## 8. 预期效果与风险

预期效果：相比原始 ADDA，新增类别级约束后，目标特征不只做全局域对齐，还会向语义正确的源类中心靠近，通常能减少类别混淆。

主要风险：伪标签错误会被自训练放大。因此默认使用高阈值、warm-up 和只对高置信度样本做原型对齐。若目标域初始预测很差，可提高 `--regularizer-start-epoch`、降低相关权重，或先使用更长源域预训练。
