# ADDA 与 CDAN 算法解释及其与“单源域自适应图像分类算法设计与实现”的关联

本文基于当前项目源码，对 `ADDA` 与 `CDAN` 两个域自适应图像分类算法进行整理说明，并解释它们为什么适合作为毕业设计题目 **“单源域自适应图像分类算法设计与实现”** 的核心算法或对比算法。

## 2. 域自适应图像分类问题背景

### 2.1 什么是“域”

在图像分类中，一个“域”可以理解为一类数据分布。例如：

- `Office31` 中的 Amazon、Webcam、DSLR 是不同域；
- `VisDA2017` 中的 Synthetic 与 Real 是不同域；
- 手写数字任务中 MNIST、USPS、SVHN 是不同域；
- 真实业务中，白天图片、夜晚图片，不同相机拍摄图片，不同医院或不同设备采集的医学图像，也可以看作不同域。

即使类别空间相同，不同域之间仍可能存在明显的分布差异。例如都是“键盘、鼠标、杯子”等类别，但 Amazon 商品图和 Webcam 实拍图在背景、光照、清晰度、拍摄角度、纹理上都不同。这种差异称为 **domain shift**。

### 2.2 单源域自适应图像分类的数学形式

毕业设计题目中的“单源域自适应图像分类”通常对应 **Single-Source Unsupervised Domain Adaptation，单源无监督域自适应**：

- 源域：有标签数据

\[
\mathcal{D}_s = \{(x_i^s, y_i^s)\}_{i=1}^{n_s}
\]

- 目标域：无标签训练数据

\[
\mathcal{D}_t = \{x_j^t\}_{j=1}^{n_t}
\]

- 源域和目标域共享同一类别空间：

\[
\mathcal{Y}_s = \mathcal{Y}_t
\]

- 但数据分布不同：

\[
P_s(x, y) \neq P_t(x, y)
\]

目标是在没有目标域训练标签的情况下，利用源域有标签数据和目标域无标签数据，训练出一个在目标域测试集上分类准确率较高的模型。

### 2.3 为什么不能只做普通监督学习

如果只在源域上训练一个普通分类器，即 ERM / Source Only：

\[
\min_{F,C} \mathbb{E}_{(x_s,y_s) \sim \mathcal{D}_s} \operatorname{CE}(C(F(x_s)), y_s)
\]

其中：

- \(F\)：特征提取器，例如 ResNet、LeNet、ViT；
- \(C\)：分类头；
- \(\operatorname{CE}\)：交叉熵损失。

这个模型会很好地拟合源域，但在目标域上可能下降明显。原因是模型学习到的特征可能包含大量源域特有信息，例如背景、纹理、拍摄方式，而不是纯粹的类别语义。因此，域自适应算法需要在保证源域分类能力的同时，减少源域和目标域特征分布之间的差异。

### 2.4 对抗式域自适应的基本思想

ADDA 和 CDAN 都属于 **对抗式域自适应** 思路。它们借鉴 GAN 的思想，引入一个域判别器 \(D\)：

- \(D\) 的任务：判断某个特征来自源域还是目标域；
- 特征提取器 \(F\) 的任务：生成让 \(D\) 分不清来源的域不变特征。

如果训练成功，域判别器无法区分源域特征和目标域特征，说明两个域在特征空间中被对齐。此时，源域上学到的分类边界更有可能迁移到目标域。

## 3. ADDA 算法思想详解

### 3.1 ADDA 的全称和核心目标

ADDA 的全称是 **Adversarial Discriminative Domain Adaptation**，中文可译为 **对抗判别式域自适应**。

ADDA 的目标是：

> 先用有标签源域训练一个可靠的源域分类模型，再通过对抗训练让目标域特征分布向源域特征分布靠近，从而把源域分类器迁移到目标域。

ADDA 的核心不是直接生成图片，也不是显式给目标域打伪标签，而是学习一种 **目标域特征映射**，使目标域图像经过特征提取后，看起来像源域特征。

### 3.2 ADDA 的网络组成

经典 ADDA 包含四个概念模块：

1. **源域特征提取器** \(F_s\)：把源域图像映射为源域特征；
2. **目标域特征提取器** \(F_t\)：把目标域图像映射为目标域特征；
3. **分类器** \(C\)：根据特征输出类别预测；
4. **域判别器** \(D\)：判断输入特征来自源域还是目标域。

在项目实现中：

- `source_classifier` 对应源域模型；
- `target_classifier` 由 `source_classifier` 深拷贝得到，对应目标域模型；
- `DomainDiscriminator` 是域判别器；
- `ImageClassifier` 由 `backbone + pool_layer + bottleneck + head` 组成。

`ImageClassifier` 的基本结构可以概括为：

\[
x \rightarrow \text{Backbone} \rightarrow \text{Pooling} \rightarrow \text{Bottleneck} \rightarrow \text{Classifier Head}
\]

其中 `bottleneck` 在 `tllib/alignment/adda.py` 中定义为：

```python
nn.Linear(backbone.out_features, bottleneck_dim)
nn.BatchNorm1d(bottleneck_dim)
nn.ReLU()
```

它的作用是把骨干网络输出的高维图像特征压缩到一个适合域对齐的特征空间中，默认维度为 256。

### 3.3 ADDA 的两阶段训练流程

ADDA 的典型训练过程分为两个阶段。

#### 阶段一：源域监督预训练

第一阶段只使用有标签源域数据训练源域分类器：

\[
\min_{F_s,C} \mathcal{L}_{cls}
= \mathbb{E}_{(x_s,y_s) \sim \mathcal{D}_s}
\operatorname{CE}(C(F_s(x_s)), y_s)
\]

目标是让模型在源域上具备较强分类能力。

对应到项目 `adda.py`：

```python
utils.empirical_risk_minimization(
    train_source_iter,
    pretrain_model,
    pretrain_optimizer,
    pretrain_lr_scheduler,
    epoch,
    args,
    device
)
```

如果用户没有通过 `--pretrain` 指定预训练权重，代码会先在源域上训练 `pretrain_model`，然后保存为 `pretrain` checkpoint。

#### 阶段二：目标域对抗适配

第二阶段加载源域预训练权重，并创建两个模型：

```python
source_classifier.load_state_dict(checkpoint)
target_classifier = copy.deepcopy(source_classifier)
```

然后固定源域模型：

```python
set_requires_grad(source_classifier, False)
source_classifier.freeze_bn()
```

固定源域模型的意义是：源域模型已经定义了一个“源域特征空间”和分类边界，后续要做的是让目标域特征靠近这个空间，而不是让源域空间跟着目标域一起漂移。

在每个训练 iteration 中：

```python
_, f_s = source_model(x_s)
_, f_t = target_model(x_t)
loss_transfer = domain_adv(f_s, f_t)
```

其中：

- \(f_s = F_s(x_s)\)：源域特征；
- \(f_t = F_t(x_t)\)：目标域特征；
- `domain_adv(f_s, f_t)`：域对抗损失。

域判别器的目标是区分源域和目标域：

\[
D(f_s) \rightarrow 1, \quad D(f_t) \rightarrow 0
\]

其中项目公共模块 `DomainDiscriminator` 默认设定：

- 源域标签为 1；
- 目标域标签为 0；
- 输出为一个 sigmoid 概率。

域判别器损失可以写成：

\[
\mathcal{L}_D =
-\mathbb{E}_{x_s \sim \mathcal{D}_s}\log D(F_s(x_s))
-\mathbb{E}_{x_t \sim \mathcal{D}_t}\log(1-D(F_t(x_t)))
\]

而目标特征提取器的目标正好相反：它希望目标域特征骗过判别器，让判别器把目标域也判断为源域。经典 ADDA 常使用 GAN 中的非饱和目标：

\[
\mathcal{L}_{F_t} =
-\mathbb{E}_{x_t \sim \mathcal{D}_t}\log D(F_t(x_t))
\]

直观解释：

- 判别器越能分清源域和目标域，说明两个域的特征差异越大；
- 目标特征提取器越能欺骗判别器，说明目标特征越像源域特征；
- 当判别器只能接近随机猜测时，说明源域和目标域在特征空间上已经较好对齐。

### 3.4 项目中 ADDA 采用的梯度反转实现

项目 `adda.py` 中创建了梯度反转层：

```python
grl = WarmStartGradientReverseLayer(alpha=1., lo=0., hi=2., max_iters=1000, auto_step=True)
domain_adv = DomainAdversarialLoss(domain_discri, grl=grl).to(device)
```

梯度反转层 GRL 的前向传播是恒等映射：

\[
\operatorname{GRL}(f) = f
\]

但反向传播时会把梯度乘以负系数：

\[
\frac{\partial \operatorname{GRL}(f)}{\partial f} = -\lambda I
\]

这样可以在一次反向传播中同时完成两个方向的优化：

- 域判别器 \(D\)：正常最小化域分类损失，努力区分源域和目标域；
- 目标特征提取器 \(F_t\)：收到反向后的梯度，努力让域判别器分不清。

`WarmStartGradientReverseLayer` 的 “WarmStart” 表示反转强度 \(\lambda\) 会逐渐变化，而不是训练一开始就使用很强的对抗信号。这样通常可以减少训练初期的不稳定，因为模型还没有学到可靠特征时，过强的域对抗可能破坏分类语义。

### 3.5 ADDA 的优化对象

项目中 ADDA 的优化器为：

```python
optimizer = SGD(
    target_classifier.get_parameters(optimize_head=False) + domain_discri.get_parameters(),
    args.lr,
    momentum=args.momentum,
    weight_decay=args.weight_decay,
    nesterov=True
)
```

这意味着适配阶段主要更新：

1. 目标域特征提取器的 backbone；
2. 目标域 bottleneck；
3. 域判别器。

分类头没有被加入优化器。这样做符合 ADDA 的基本思想：源域分类器已经通过有标签数据训练好，目标域适配阶段重点是把目标域特征移动到源域分类边界可以正确工作的区域。

### 3.6 ADDA 的算法流程总结

可以把项目中的 ADDA 流程概括为：

1. 准备源域有标签数据和目标域无标签数据；
2. 使用源域标签训练源域分类模型；
3. 复制源域模型得到目标域模型；
4. 固定源域模型；
5. 构造域判别器；
6. 每次取一批源域图像和目标域图像；
7. 用源模型提取源域特征，用目标模型提取目标域特征；
8. 域判别器学习区分源 / 目标；
9. 目标特征提取器通过对抗梯度学习欺骗判别器；
10. 在验证集和测试集上评估目标域分类准确率。

### 3.7 ADDA 的优点

1. **思想清晰，结构简单**：源域训练和目标域适配分成两个阶段，便于实现和论文阐述。
2. **不需要目标域标签**：目标域训练阶段只需要目标域图片，不需要类别标注。
3. **适合做基线算法**：ADDA 是经典对抗式域自适应方法，适合作为毕业设计中的核心基线。
4. **能够利用成熟分类骨干网络**：项目中可选择 `resnet18`、`resnet50`、`resnet101`、ViT、LeNet 等 backbone。

### 3.8 ADDA 的局限

ADDA 主要对齐的是整体特征分布，即边缘分布：

\[
P_s(f) \approx P_t(f)
\]

但图像分类是多类别问题，不同类别对应不同语义簇。如果只做整体对齐，可能出现以下问题：

- 目标域的“猫”特征被错误对齐到源域“狗”附近；
- 类别边界附近的目标样本被对齐到错误类别区域；
- 源域和目标域整体看起来接近，但类别条件分布仍不一致。

也就是说，ADDA 关注“这个特征像不像源域”，但没有显式关注“这个特征像源域中的哪个类别”。CDAN 正是针对这个不足提出了改进。

## 4. CDAN 算法思想详解

### 4.1 CDAN 的全称和核心目标

CDAN 的全称是 **Conditional Domain Adversarial Network**，中文可译为 **条件域对抗网络** 或 **条件对抗域自适应网络**。

CDAN 的核心思想是：

> 域对齐不应该只看图像特征本身，还应该结合分类器对该样本的类别预测。也就是让域判别器在“类别条件”下判断源域和目标域，从而实现更细粒度的类条件对齐。

相比 ADDA / DANN 这类只对齐特征 \(f\) 的方法，CDAN 对齐的是特征和分类预测之间的联合信息：

\[
(f, g)
\]

其中：

- \(f = F(x)\)：特征提取器输出的图像特征；
- \(g = C(f)\)：分类器输出的预测 logits 或 softmax 概率。

### 4.2 为什么 CDAN 要引入分类条件

普通域对抗方法通常训练域判别器：

\[
D(f) \rightarrow \{\text{source}, \text{target}\}
\]

它只根据特征 \(f\) 判断域来源。这相当于对齐边缘分布：

\[
P_s(f) \approx P_t(f)
\]

但是图像分类更关心每个类别内部是否对齐，即：

\[
P_s(f \mid y=c) \approx P_t(f \mid y=c)
\]

由于目标域没有标签 \(y_t\)，CDAN 使用分类器预测 \(g_t\) 作为目标域样本的类别语义信息。这样域判别器看到的不只是“一个特征”，而是“一个带有类别倾向的特征”。

例如：

- 如果某个目标域样本被模型预测为“键盘”，那么它应该和源域中同样被预测为“键盘”的特征对齐；
- 如果只做整体对齐，则“键盘”有可能被拉向“鼠标”或“显示器”的源域特征簇；
- CDAN 通过引入预测分布，让对齐过程更关注类别结构。

### 4.3 CDAN 的网络组成

项目中的 CDAN 主要包含：

1. **特征提取器 + 分类器**：`ImageClassifier`；
2. **域判别器**：`DomainDiscriminator`；
3. **条件映射函数**：`MultiLinearMap` 或 `RandomizedMultiLinearMap`；
4. **条件域对抗损失**：`ConditionalDomainAdversarialLoss`。

在 `examples/domain_adaptation/image_classification/cdan.py` 中：

```python
classifier = ImageClassifier(backbone, num_classes, bottleneck_dim=args.bottleneck_dim, ...)
```

若不使用随机映射，域判别器输入维度为：

```python
classifier_feature_dim * num_classes
```

也就是特征维度乘以类别数。这来自 CDAN 的多线性映射。

### 4.4 多线性映射：把特征和类别预测结合起来

CDAN 的关键是构造一个联合表示：

\[
T(f,g)
\]

项目中 `MultiLinearMap` 的实现为：

```python
output = torch.bmm(g.unsqueeze(2), f.unsqueeze(1))
return output.view(batch_size, -1)
```

这等价于对每个样本计算类别预测 \(g\) 和特征 \(f\) 的外积：

\[
T(f,g) = g \otimes f
\]

如果：

- \(f \in \mathbb{R}^F\)；
- \(g \in \mathbb{R}^C\)；

那么：

\[
T(f,g) \in \mathbb{R}^{C \times F}
\]

展平后输入域判别器，维度为 \(C \cdot F\)。

这个外积有一个很重要的直观意义：

- 如果某个样本预测为第 1 类概率高，那么第 1 类对应的特征块权重大；
- 如果某个样本预测为第 2 类概率高，那么第 2 类对应的特征块权重大；
- 因此域判别器实际上是在不同类别条件下判断源域和目标域是否一致。

也就是说，CDAN 不再简单问：

> 这个特征来自源域还是目标域？

而是问：

> 对于当前类别预测结构，这个“类别相关特征”来自源域还是目标域？

### 4.5 随机多线性映射：降低计算和显存开销

如果类别数 \(C\) 和特征维度 \(F\) 都较大，直接使用外积 \(C \times F\) 会导致域判别器输入维度很高。为此 CDAN 提供 `RandomizedMultiLinearMap`：

\[
T_{\odot}(f,g) = \frac{1}{\sqrt{d}}(R_f f) \odot (R_g g)
\]

其中：

- \(R_f\)：固定随机矩阵，把特征映射到 \(d\) 维；
- \(R_g\)：固定随机矩阵，把类别预测映射到 \(d\) 维；
- \(\odot\)：逐元素乘法；
- \(d\)：随机映射后的维度，由 `--randomized-dim` 控制。

项目中对应代码：

```python
f = torch.mm(f, self.Rf.to(f.device))
g = torch.mm(g, self.Rg.to(g.device))
output = torch.mul(f, g) / np.sqrt(float(self.output_dim))
```

在 `cdan.sh` 的 DomainNet 实验中，可以看到使用 `-r -rd 51200`，说明大规模类别或高维特征场景下，随机多线性映射有助于控制显存和计算成本。

### 4.6 CDAN 的训练损失

CDAN 的总损失由两部分组成。

#### 1. 源域分类损失

源域有标签，因此仍然使用交叉熵训练分类器：

\[
\mathcal{L}_{cls}
= \mathbb{E}_{(x_s,y_s) \sim \mathcal{D}_s}
\operatorname{CE}(C(F(x_s)), y_s)
\]

项目中：

```python
cls_loss = F.cross_entropy(y_s, labels_s)
```

#### 2. 条件域对抗损失

域判别器不再直接输入 \(f\)，而是输入 \(T(f,g)\)：

\[
\mathcal{L}_{adv}
= -\mathbb{E}_{x_s \sim \mathcal{D}_s}\log D(T(f_s,g_s))
-\mathbb{E}_{x_t \sim \mathcal{D}_t}\log(1-D(T(f_t,g_t)))
\]

项目中：

```python
transfer_loss = domain_adv(y_s, f_s, y_t, f_t)
```

最终总损失：

\[
\mathcal{L}
= \mathcal{L}_{cls} + \lambda \mathcal{L}_{adv}
\]

项目中：

```python
loss = cls_loss + transfer_loss * args.trade_off
```

其中 `args.trade_off` 就是权重系数 \(\lambda\)，默认值为 1。

### 4.7 CDAN 中的梯度反转层

`ConditionalDomainAdversarialLoss` 内部创建了梯度反转层：

```python
self.grl = WarmStartGradientReverseLayer(alpha=1., lo=0., hi=1., max_iters=1000, auto_step=True)
```

前向计算时：

```python
h = self.grl(self.map(f, g))
d = self.domain_discriminator(h)
```

这表示域判别器基于条件表示 \(T(f,g)\) 预测域标签，但特征提取器和分类器在反向传播时会收到反向梯度，促使它们生成域不可区分的条件表示。

### 4.8 CDAN 中的 softmax 和 detach

在 `ConditionalDomainAdversarialLoss.forward()` 中：

```python
g = F.softmax(g, dim=1).detach()
```

这说明传入的 `g_s`、`g_t` 原本是分类器 logits，CDAN 会先把它们转成类别概率分布，然后 `detach`。

这样做的含义是：

1. 域判别器使用分类预测概率作为条件信息；
2. `detach` 阻止域对抗损失直接通过预测概率分支强行改变分类器输出，避免对分类决策造成过度扰动；
3. 域对抗主要通过条件表示中的特征部分影响特征提取器，使特征分布对齐。

### 4.9 熵条件加权：更重视高置信度样本

CDAN 还支持 entropy conditioning，即熵条件加权。项目中通过参数 `--entropy` 开启。

预测分布的熵为：

\[
H(g) = -\sum_{c=1}^{C} g_c \log g_c
\]

如果一个样本预测很确定，例如某一类概率接近 1，则熵较低；如果预测很不确定，各类概率接近平均，则熵较高。

项目中权重为：

```python
weight = 1.0 + torch.exp(-entropy(g))
weight = weight / torch.sum(weight) * batch_size
```

低熵样本权重大，高熵样本权重小。直观上：

- 高置信度样本的类别语义更可靠，适合用于条件对齐；
- 低置信度样本可能位于类别边界附近或预测错误，过度对齐可能带来负迁移。

因此熵条件加权可以让模型更关注“预测可靠”的源域 / 目标域样本。

### 4.10 CDAN 的训练流程总结

项目中的 CDAN 每个 iteration 的流程为：

1. 从源域取有标签 batch：\((x_s, y_s)\)；
2. 从目标域取无标签 batch：\(x_t\)；
3. 拼接源域和目标域图像：

   ```python
   x = torch.cat((x_s, x_t), dim=0)
   ```

4. 通过分类模型得到 logits 和 features：

   ```python
   y, f = model(x)
   y_s, y_t = y.chunk(2, dim=0)
   f_s, f_t = f.chunk(2, dim=0)
   ```

5. 对源域计算分类损失：

   ```python
   cls_loss = F.cross_entropy(y_s, labels_s)
   ```

6. 对源域和目标域计算条件域对抗损失：

   ```python
   transfer_loss = domain_adv(y_s, f_s, y_t, f_t)
   ```

7. 加权求和得到总损失：

   ```python
   loss = cls_loss + transfer_loss * args.trade_off
   ```

8. 更新分类器、特征提取器和域判别器；
9. 在验证集和测试集上评估目标域准确率。

### 4.11 CDAN 的优点

1. **类条件对齐能力更强**：相比只对齐整体特征分布，CDAN 通过 \(T(f,g)\) 更关注类别结构。
2. **更适合多类别图像分类**：图像分类任务通常是多峰分布，不同类别天然形成不同簇，条件对齐更合理。
3. **仍然不需要目标域标签**：目标域只需参与对抗训练，不需要人工标注。
4. **可扩展性较好**：通过随机多线性映射，可以适配类别数较多、特征维度较高的数据集。
5. **可以与熵加权结合**：进一步降低不确定目标样本对训练的负面影响。

### 4.12 CDAN 的局限

1. **依赖分类器预测质量**：如果训练早期目标域预测很差，条件信息 \(g_t\) 可能不可靠。
2. **计算量更大**：普通多线性映射维度为 \(C \times F\)，类别数多时显存开销明显。
3. **对抗训练仍可能不稳定**：和 ADDA 一样，域判别器过强或过弱都会影响适配效果。
4. **不直接保证目标域伪标签正确**：CDAN 使用预测分布作为条件，但没有显式纠正错误预测。

## 5. ADDA 与 CDAN 的对比

| 对比维度 | ADDA | CDAN |
| --- | --- | --- |
| 全称 | Adversarial Discriminative Domain Adaptation | Conditional Domain Adversarial Network |
| 基本思想 | 让目标域特征整体对齐源域特征 | 结合类别预测，让源/目标在类别条件下对齐 |
| 域判别器输入 | 特征 \(f\) | 条件表示 \(T(f,g)\)，即特征 + 分类预测 |
| 主要对齐对象 | 边缘特征分布 \(P(f)\) | 联合 / 条件分布 \(P(f,g)\) 或近似 \(P(f \mid y)\) |
| 是否需要目标标签 | 不需要 | 不需要 |
| 训练方式 | 源域预训练 + 目标域对抗适配 | 源域分类损失 + 条件域对抗损失联合训练 |
| 优点 | 简单直观，适合作为基线 | 类别感知更强，通常更适合多类别分类 |
| 风险 | 可能发生类别错配 | 依赖预测质量，计算更重 |
| 项目入口 | `adda.py` | `cdan.py` |

从算法发展逻辑看，可以把 ADDA 理解为“先解决域整体对齐”，把 CDAN 理解为“进一步解决分类条件下的细粒度对齐”。因此在毕业设计中，两者非常适合形成递进关系：

1. 先介绍普通源域监督模型 ERM 的不足；
2. 再介绍 ADDA 使用对抗训练进行域不变特征学习；
3. 最后介绍 CDAN 在 ADDA / DANN 思想上加入类别条件信息，改善类别级对齐。

## 6. 两个算法与毕业设计题目的关联

毕业设计题目是：

> 单源域自适应图像分类算法设计与实现

这个题目包含几个关键词：

1. **单源域**：训练时只有一个带标签源域；
2. **自适应**：模型需要适应一个分布不同的目标域；
3. **图像分类**：最终任务是对目标域图像进行类别预测；
4. **算法设计与实现**：不仅要解释理论，还要有可运行代码、训练流程和实验验证。

ADDA 和 CDAN 与该题目高度匹配。

### 6.1 单源域设置匹配

项目脚本中有大量单源域命令，例如 `adda.sh` 和 `cdan.sh`：

```bash
python adda.py data/office31 -d Office31 -s A -t W -a resnet50 ...
python cdan.py data/office31 -d Office31 -s A -t W -a resnet50 ...
```

其中：

- `-s A` 表示源域为 Amazon；
- `-t W` 表示目标域为 Webcam；
- 训练时源域有标签，目标域训练数据不使用标签；
- 测试时在目标域上计算分类准确率。

这正是单源域自适应图像分类的典型实验形式。

虽然脚本中也包含多源域实验，例如 `-s Cl Pr Rw -t Ar`，但毕业设计可以只采用 `-s` 后面一个源域的命令，即单源设置。

### 6.2 图像分类任务匹配

两个算法都围绕图像分类模型构建：

- 输入是图像；
- backbone 提取视觉特征；
- classifier head 输出类别；
- 评价指标是目标域分类准确率；
- 支持 Office31、OfficeHome、VisDA、DomainNet、Digits 等图像分类域自适应数据集。

这与“图像分类算法设计与实现”完全一致。

### 6.3 域自适应方法匹配

ADDA 和 CDAN 都显式解决源域和目标域分布不一致的问题：

- ADDA 通过目标域特征对齐源域特征，学习域不变表示；
- CDAN 通过条件域对抗，使同类别结构下的源域和目标域更一致。

因此它们不是普通分类算法，而是专门的域自适应图像分类算法。

### 6.4 适合作为论文中的“基线 + 改进”结构

毕业设计可以这样组织：

1. **Source Only / ERM**：只在源域训练，不做域适配，作为最基础基线；
2. **ADDA**：引入对抗式域适配，对齐源 / 目标整体特征分布；
3. **CDAN**：在对抗式域适配基础上加入分类条件，实现更细粒度的类别对齐。

这种结构有清晰的递进关系：

\[
\text{Source Only} \rightarrow \text{ADDA} \rightarrow \text{CDAN}
\]

对应的论文叙述也很自然：

- Source Only 说明 domain shift 问题；
- ADDA 说明对抗式域不变特征学习；
- CDAN 说明仅对齐边缘分布不足，需要利用类别预测进行条件对齐。

### 6.5 适合实现层面的模块化设计

在项目实现中，两个算法共用大量模块：

- 数据加载：`utils.get_dataset()`；
- 图像变换：`utils.get_train_transform()`、`utils.get_val_transform()`；
- 分类器结构：`ImageClassifier`；
- 域判别器：`DomainDiscriminator`；
- 训练日志和 checkpoint：`CompleteLogger`；
- 可视化分析：t-SNE、A-distance。

这有利于毕业设计中的“系统实现”部分。可以把系统分成：

1. 数据处理模块；
2. 特征提取与分类模块；
3. 域判别模块；
4. 对抗训练模块；
5. 评估与可视化模块。

### 6.6 适合实验分析

项目中提供了以下评估方式：

1. **目标域 Top-1 Accuracy**：主要分类指标；
2. **Per-class Accuracy**：类别级准确率，适用于 VisDA 等类别不均衡任务；
3. **Domain Discriminator Accuracy**：域判别器准确率，可以观察对齐程度；
4. **t-SNE 可视化**：观察源域 / 目标域特征是否混合；
5. **A-distance**：衡量源域和目标域特征分布差异。

这些指标和分析方式可以支撑毕业论文实验部分，例如：

- 比较 Source Only、ADDA、CDAN 在不同迁移方向上的准确率；
- 分析 CDAN 是否比 ADDA 更能保持类别结构；
- 用 t-SNE 展示适配前后特征分布变化；
- 用 A-distance 说明域间差异是否减小。

## 7. 可写入毕业论文的算法描述框架

下面给出一个可以直接转化为论文内容的写作框架。

### 7.1 问题定义

设源域 \(\mathcal{D}_s\) 包含有标签样本，目标域 \(\mathcal{D}_t\) 仅包含无标签样本。源域和目标域共享类别空间，但数据分布不同。单源域自适应图像分类的目标是在不使用目标域训练标签的条件下，学习一个能够在目标域上取得良好分类性能的模型。

### 7.2 ADDA 方法描述

ADDA 首先在源域有标签数据上训练源域特征提取器和分类器，使其具备基本图像分类能力。随后复制源域模型得到目标域特征提取器，并固定源域模型。训练过程中，引入域判别器判断特征来自源域还是目标域。域判别器学习区分两域，目标域特征提取器则学习欺骗域判别器，使目标域特征分布逐渐靠近源域特征分布。最终，目标域图像经过适配后的目标特征提取器和源域分类头完成分类。

### 7.3 CDAN 方法描述

CDAN 认为仅对齐特征边缘分布不足以保证分类性能，因为不同类别的特征分布具有多峰结构。为此，CDAN 将图像特征与分类器预测结果进行多线性融合，构造条件表示，并把该条件表示输入域判别器。域判别器在类别预测条件下判断样本来自源域还是目标域，特征提取器则通过梯度反转层学习域不可区分的条件表示。该方法能够更好地对齐同类别下的源域和目标域特征，从而提升目标域分类效果。

## 8. 实验设计建议

结合当前项目，毕业设计实验可以采用以下方案。

### 8.1 数据集选择

优先选择代码已支持的数据集：

1. `Office31`：规模较小，适合快速验证；
2. `OfficeHome`：类别和域更多，适合更完整实验；
3. `Digits`：MNIST、USPS、SVHN，训练成本较低；
4. `VisDA2017`：合成到真实迁移，难度较高。

如果时间有限，建议使用：

- `Office31 A -> W`、`D -> A`、`W -> A`；
- 或 `Digits MNIST -> USPS`、`USPS -> MNIST`。

### 8.2 对比方法

建议至少包含：

| 方法 | 作用 |
| --- | --- |
| Source Only / ERM | 不做域适配的基础模型 |
| ADDA | 对抗式边缘分布对齐方法 |
| CDAN | 条件域对抗方法 |

如果论文篇幅允许，也可以加入 DANN 作为中间对比，因为当前 ADDA 实现中也使用了 DANN 风格的 GRL 域对抗损失。

### 8.3 消融实验

可以围绕 CDAN 做消融：

1. 是否使用 `--randomized`；
2. 是否使用 `--entropy`；
3. 不同 `--trade-off` 权重；
4. 不同 backbone，如 `resnet18` 与 `resnet50`；
5. 不同 bottleneck 维度。

### 8.4 可视化分析

项目提供 `--phase analysis`，可以输出：

- t-SNE 可视化图；
- A-distance。

论文中可以展示：

1. Source Only 特征分布：源域和目标域分离明显；
2. ADDA 特征分布：两域整体更混合；
3. CDAN 特征分布：同类别聚类更清晰，跨域混合更合理。

## 9. 总结

ADDA 和 CDAN 都是典型的无监督域自适应图像分类算法，非常适合支撑“单源域自适应图像分类算法设计与实现”这一毕业设计题目。

- **ADDA** 的重点是通过对抗训练让目标域特征靠近源域特征，是经典的两阶段对抗式域自适应方法。它结构清晰、实现直观，适合作为核心基线。
- **CDAN** 的重点是将类别预测作为条件信息，引导域判别器进行类别感知的对抗训练。它解决了只做整体特征对齐可能造成类别错配的问题，是 ADDA / DANN 思路上的进一步改进。
- 从毕业设计角度看，ADDA 可以体现“域不变特征学习”的基本思想，CDAN 可以体现“类别条件对齐”的改进思想，两者能够形成完整的算法设计、代码实现、实验对比和结果分析闭环。

因此，论文可以围绕以下主线展开：

\[
\text{源域监督分类} \rightarrow \text{对抗式域适配（ADDA）} \rightarrow \text{条件域对抗适配（CDAN）}
\]

这条主线既符合单源域自适应图像分类任务定义，也与当前项目代码结构和实验脚本高度一致。
