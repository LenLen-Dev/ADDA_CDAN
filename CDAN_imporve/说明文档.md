# TLC-CDAN：温度调节的可学习紧凑条件域对抗网络说明文档

本目录实现的是对原始 CDAN（Conditional Domain Adversarial Network）的一个改进版本。为了避免与你已经在 DANN、DAAN 中使用过的伪标签、熵最小化、类原型对齐等思想重复，本改进只围绕 CDAN 最核心的机制——**条件映射 `T(f, g)`** 展开。

目录名保持为用户指定的 `CDAN_imporve`。

---

## 1. 改进动机

原始 CDAN 的核心思想是把图像特征 `f` 和分类预测 `g` 组合成条件表示，然后送入域判别器：

\[
T(f,g)=g\otimes f
\]

其中：

- \(f\)：特征提取器输出的图像特征；
- \(g\)：分类器预测概率；
- \(g\otimes f\)：类别预测和图像特征的外积。

这种设计比 DANN 只输入特征 `f` 更细，因为域判别器可以在类别预测条件下判断源域和目标域是否对齐。但是它也有三个问题：

1. **外积维度高**：如果特征维度为 \(F\)，类别数为 \(C\)，域判别器输入维度为 \(F\times C\)。例如 `F=256, C=31` 时输入维度为 `7936`。
2. **随机映射不可学习**：原始 CDAN 的随机多线性映射能降维，但随机矩阵固定，不能根据当前数据集学习更合适的条件交互。
3. **预测条件早期不稳定**：目标域没有标签，训练早期分类预测可能较不可靠；如果直接使用过尖锐的预测分布作为条件信息，容易放大错误条件。

因此，本实现提出 **TLC-CDAN（Temperature-guided Learnable Compact CDAN）**：

> 使用可学习低维条件映射替代高维外积或固定随机映射，并使用温度调节的类别概率作为条件向量，提高条件域对抗训练的稳定性和可实现性。

## 2. 算法创新点

### 2.1 可学习紧凑条件映射

原始 CDAN：

\[
T(f,g)=g\otimes f
\]

本方法改为：

\[
z_f=\phi_f(f)
\]

\[
z_g=\phi_g(g)
\]

\[
T_\theta(f,g)=\operatorname{Norm}\left(\frac{z_f\odot z_g}{\sqrt d}\right)
\]

其中：

- \(\phi_f\)：可学习的特征投影分支；
- \(\phi_g\)：可学习的类别条件投影分支；
- \(\odot\)：逐元素乘法；
- \(d\)：紧凑条件表示维度，由 `--condition-dim` 控制；
- `Norm`：LayerNorm，用于稳定融合后的条件表示。

代码对应 `lc_cdan.py` 中的：

```python
LearnableCompactConditionalMap
```

这样做的好处是：

- 把原来的 `features_dim * num_classes` 维输入压缩为固定 `condition_dim` 维；
- 条件交互方式由模型学习，而不是固定随机矩阵；
- 仍然保留 CDAN “特征 + 类别预测”联合对齐的核心思想。

### 2.2 温度调节类别条件向量

原始 CDAN 使用：

\[
g=softmax(y)
\]

本方法使用：

\[
g_T=softmax(y/T)
\]

其中 `T` 是温度系数：

- `T` 较大：预测分布更平滑，适合训练早期，减少错误尖锐预测对条件对齐的干扰；
- `T` 较小：预测分布更明确，适合训练后期，增强类别条件的判别性。

默认采用线性调度：

\[
T(p)=T_{max}-(T_{max}-T_{min})p
\]

其中 \(p\in[0,1]\) 表示训练进度。默认参数为：

```text
--max-temperature 2.0
--min-temperature 1.0
--temperature-schedule linear
```

### 2.3 不引入重复改进思想

本实现不使用以下机制：

- 目标域伪标签学习；
- 目标域熵最小化；
- 类原型对齐；
- 源/目标类中心约束。

原因是这些思想已经被用于你的 DANN、DAAN 改进中。TLC-CDAN 的创新点集中在 CDAN 独有的条件映射机制上，因此与 DANN、DAAN 的改进方向区分明显。

## 3. 损失函数

训练仍然使用单源无监督域自适应设置：

- 源域：有标签 \((x_s,y_s)\)；
- 目标域：无标签 \(x_t\)。

源域分类损失：

\[
\mathcal{L}_{cls}^{s}=CE(C(F(x_s)),y_s)
\]

改进条件域对抗损失：

\[
\mathcal{L}_{adv}=
-\mathbb{E}_{x_s}\log D(T_\theta(f_s,g_s^T))
-\mathbb{E}_{x_t}\log(1-D(T_\theta(f_t,g_t^T)))
\]

总损失：

\[
\mathcal{L}=\mathcal{L}_{cls}^{s}+\lambda\mathcal{L}_{adv}
\]

其中 `lambda` 由命令行参数 `--trade-off` 控制。

## 5. 运行方式

### 5.1 随机张量快速验证

```bash
python -m py_compile CDAN_imporve/lc_cdan.py CDAN_imporve/cdan_improve.py CDAN_imporve/smoke_test.py
python CDAN_imporve/smoke_test.py
```

### 5.2 Office31 示例

```bash
python CDAN_imporve/cdan_improve.py examples/domain_adaptation/image_classification/data/office31 \
  -d Office31 -s A -t W -a resnet50 --epochs 20 --seed 2 \
  --condition-dim 1024 --max-temperature 2.0 --min-temperature 1.0 \
  --log logs/tlc_cdan/Office31_A2W
```

### 5.3 Digits 小规模示例

```bash
python CDAN_imporve/cdan_improve.py examples/domain_adaptation/image_classification/data/digits \
  -d Digits -s MNIST -t USPS --train-resizing res. --val-resizing res. \
  --resize-size 28 --no-hflip --norm-mean 0.5 --norm-std 0.5 \
  -a lenet --no-pool --scratch -b 32 -i 100 --epochs 5 \
  --log logs/tlc_cdan/MNIST2USPS
```

## 6. 参数说明

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| `--condition-dim` | `1024` | 可学习紧凑条件表示维度 |
| `--condition-dropout` | `0.1` | 条件映射分支中的 Dropout 概率 |
| `--discriminator-hidden-size` | `1024` | 域判别器隐藏层维度 |
| `--max-temperature` | `2.0` | 训练初期或 constant 模式下的温度 |
| `--min-temperature` | `1.0` | linear 模式下训练末期温度 |
| `--temperature-schedule` | `linear` | 温度策略，可选 `linear` 或 `constant` |
| `--trade-off` | `1.0` | 条件域对抗损失权重 |
| `--entropy` | `False` | 使用 CDAN 原有的熵感知样本加权，不是目标域熵最小化 |

## 7. 与毕业设计课题的关联

毕业设计题目为：

> 单源域自适应图像分类算法设计与实现

TLC-CDAN 与该题目高度匹配：

1. **单源域**：训练入口仍然使用一个源域 `-s A` 和一个目标域 `-t W`。
2. **无监督目标域**：目标域训练数据不使用类别标签。
3. **图像分类**：模型仍然是 `backbone + bottleneck + classifier head` 的图像分类结构。
4. **自适应机制**：通过条件域对抗损失对齐源域和目标域。
5. **算法创新**：改进点不是泛化的伪标签或原型思想，而是针对 CDAN 条件映射本身进行结构设计。

论文中可以这样概括：

> 针对原始 CDAN 多线性外积映射维度高、随机多线性映射不可学习以及训练早期目标域预测条件不稳定的问题，本文提出一种温度调节的可学习紧凑条件映射模块。该模块通过两个可学习投影分支分别编码图像特征和类别预测，并以低维逐元素乘法建模二者交互，在降低域判别器输入维度的同时保留类别条件对齐能力，从而提升单源域自适应图像分类的训练稳定性和迁移效果。

## 8. 建议实验与消融

建议至少比较：

| 方法 | 目的 |
| --- | --- |
| Source Only | 不做域适应的基础分类器 |
| DANN | 基础域对抗方法 |
| DAAN | 动态/局部域对抗对比方法 |
| CDAN | 原始条件域对抗方法 |
| TLC-CDAN | 本目录实现的改进方法 |

建议消融实验：

| 消融项 | 说明 |
| --- | --- |
| 原始外积 CDAN vs TLC-CDAN | 验证可学习紧凑条件映射是否有效 |
| 不同 `condition_dim` | 分析条件表示维度对准确率和计算量的影响 |
| `linear` vs `constant` 温度 | 验证温度调节是否有助于稳定训练 |
| 不同 `max_temperature` | 分析训练早期条件平滑强度 |

## 9. 与原始 CDAN 的主要区别

| 维度 | 原始 CDAN | TLC-CDAN |
| --- | --- | --- |
| 条件映射 | 外积 `g ⊗ f` 或固定随机映射 | 可学习低维投影 + 逐元素融合 |
| 域判别器输入维度 | `features_dim * num_classes` 或随机维度 | 固定 `condition_dim` |
| 条件向量 | `softmax(logits)` | `softmax(logits / T)` |
| 是否可学习条件交互 | 外积无参数，随机映射固定 | 可学习 |
| 是否使用目标伪标签 | 否 | 否 |
| 是否使用目标熵最小化 | 否 | 否 |
| 是否使用原型对齐 | 否 | 否 |

---

## 10. 注意事项

- 本实现为了保留原始 CDAN 的分类稳定性，对类别条件向量 `g_T` 使用了 `detach()`；这与原始项目 `tllib/alignment/cdan.py` 的实现一致。
- 训练脚本只保存分类器权重，因为测试阶段只需要分类器，不需要域判别器和条件映射模块。
- 若数据集类别数非常少，可适当降低 `--condition-dim`，例如 `256` 或 `512`。
- 若训练不稳定，可尝试增大 `--max-temperature` 或使用 `--temperature-schedule constant`。
